# コラム記事 CSVインポートフォーマット

Supabaseの `articles` テーブルへ一括インポートするためのCSVフォーマット仕様です。
ExcelやGoogleスプレッドシートで原稿を作成し、「CSV形式」でダウンロードして使用することを推奨します。

## ヘッダー項目（1行目）
`slug, title, description, content, category, tags, faq, thumbnail_url, status, published_at`

## 各カラムの入力ルール

| カラム名 | 必須 | 説明・入力例 |
| :--- | :--- | :--- |
| **slug** | 必須 | URLの末尾になるID。**半角英数字とハイフンのみ**。<br>例: `freelance-java-salary-2026` |
| **title** | 必須 | 記事のタイトル。<br>例: `フリーランスJavaエンジニアの年収相場` |
| **description** | 必須 | 記事の概要（メタディスクリプション）。100文字程度。<br>例: `Javaエンジニアの単価相場と高単価獲得のポイントを解説します。` |
| **content** | 必須 | 本文。**Markdown形式**で記述します。<br>見出しは `##` や `###` を使用。<br>Excel等のセル内改行はそのまま反映されます。 |
| **category** | 任意 | 大カテゴリ。<br>例: `Career`, `Tax`, `Skill`, `Interview` |
| **tags** | 任意 | タグ。**カンマ区切り**で入力。<br>例: `Java,年収,在宅` |
| **faq** | 任意 | よくある質問。以下の特殊書式で記述してください。<br>**書式:** `Q:質問文|A:回答文;Q:質問文2|A:回答文2`<br>※ 質問と回答の間は `|` (縦棒)<br>※ 質問セットの区切りは `;` (セミコロン) |
| **thumbnail_url** | 任意 | 画像URLがあれば入力。なければ空欄。 |
| **status** | 必須 | 公開状態。以下のいずれか。<br>`published` (公開), `draft` (下書き), `archived` (非公開) |
| **published_at** | 任意 | 公開日。<br>例: `2026-01-20 10:00:00` |

### 注意点
1. **カンマについて**: 本文中に半角カンマ `,` が含まれる場合、CSVの仕様上、そのセル全体をダブルクォーテーション `"` で囲む必要があります（Excel等は自動でやってくれます）。
2. **改行について**: 本文（content）の改行はそのまま有効になります。
3. **FAQの区切り文字**: 本文やFAQ内で `|` や `;` を使う場合は、全角文字にするなどして区切り文字と重複しないように注意してください。
